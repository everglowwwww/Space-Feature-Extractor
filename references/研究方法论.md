> 版本: v1.0 | 日期: 2025-06-28 | 作者: lipeilin06
> 用途: 记录《共享办公型室内空间陈设的脑认知与智能交互研究》中，从真实案例到 VR 实验空间的完整方法论链条
> 定位: 研究设计层面的方法论文档，与 PRD（技术实现）和 prompt 模板（执行规范）并列

## 一、核心逻辑链条

本研究的数据采集与实验空间构建，遵循以下完整逻辑链条：

**案例收集 → 平面图拆分 → 特征提取 → 壳子参数聚类 → 得到典型壳子原型 → VR 中构建壳子 → 在壳子内做陈设实验**

每一步的输入和输出是明确的：

| 步骤 | 输入 | 输出 | 工具/方法 |
|------|------|------|-----------|
| 案例收集 | 实地考察/网络素材 | 整层平面图 + 人视角透视照 | 手绘/拍照 |
| 平面图拆分 | 整层平面图 | 各空间单元的独立平面图 + 元信息 | plan_splitter.py |
| 特征提取 | 单元平面图 + 透视照 | 56 维特征向量 (features.json) | space-feature-extractor skill |
| 壳子参数聚类 | 多案例的壳子参数子集 | 若干种典型壳子原型 | 统计聚类（K-means 等） |
| VR 构建 | 典型壳子原型参数 | VR 实验空间 | VR 建模工具 |
| 陈设实验 | VR 壳子 + 不同陈设方案 | 被试脑电/行为数据 | EEG + VR 实验 |

## 二、壳子与陈设的分离

### 2.1 为什么要分离

共享办公空间的多样性体现在两个层面：一是空间本身的物理形态（大小、高矮、开合），二是空间内的陈设方案（家具、色彩、材质、灯光）。这两个层面在研究中扮演不同角色：

- **壳子（Shell）**：空间的物理围护条件，包括面积、层高、长宽比、围合度等。这些参数一旦建筑建成就基本固定，不随运营方的装修风格变化。在实验设计中，壳子属于**控制变量**。
- **陈设（Furnishing）**：空间内的家具布局、色彩搭配、材质选择、灯光设计等。这些是运营方可以自由调整的要素，也是本研究的核心关注对象。在实验设计中，陈设属于**实验变量（自变量）**。

分离的核心目的是确保实验的内部效度——当我们改变陈设方案来观察被试反应差异时，必须确保壳子条件保持一致，否则无法判断被试反应的变化是由陈设引起的还是由空间形态引起的。

### 2.2 壳子参数有哪些

从 56 维特征中，属于壳子参数的核心字段包括：

| 参数 | 字段名 | 说明 |
|------|--------|------|
| 净使用面积 | net_area_m2 | 去除墙体后的可用地面面积 |
| 层高 | ceiling_height_m | 地面到天花板净高 |
| 长宽比 | length_width_ratio | 空间形态，1.0=方正，>2.5=狭长 |
| 围合度 | enclosure_ratio | 空间封闭程度，0=全开放，1=全封闭 |
| 高宽比 | height_width_ratio | 空间纵向比例感 |

这些参数将用于聚类分析，从 40 个左右的空间单元案例中提取出若干种典型的壳子原型（如"小型封闭型""中型半开放型""大型开放型"等）。

### 2.3 陈设参数有哪些

属于陈设层面的特征（在实验阶段作为自变量操控）包括：

- 家具配置：家具类型、数量、密度、座位密度、材质
- 色彩与材质：主色调、色温、色彩方案、地/墙/天花材质
- 光环境：灯具类型、照度、采光系数

### 2.4 聚类分析与原型提取

聚类的目的是从多样的真实案例中，找出几种"典型的壳子形态"，作为 VR 实验空间的物理载体。

聚类的输入是壳子参数子集（面积、层高、长宽比、围合度等），不包含陈设参数。聚类方法初步考虑 K-means 或层次聚类，具体的聚类数量由轮廓系数等指标辅助确定。

聚类的输出是若干个簇心（cluster centroid），每个簇心代表一种典型壳子原型。例如，聚类可能告诉我们存在这样几种典型空间：

- 原型 A：25m²、3.0m 层高、长宽比 1.2、围合度 0.9（小型封闭办公室）
- 原型 B：60m²、3.5m 层高、长宽比 1.8、围合度 0.6（中型半开放协作区）
- 原型 C：120m²、4.0m 层高、长宽比 2.5、围合度 0.3（大型开放工位区）

这些原型将在 VR 中建模，然后在每个原型内放入不同的陈设方案进行实验。

## 三、平面图拆分与空间单元定义

### 3.1 为什么需要拆分

共享办公空间的整层平面图通常包含多种功能区域：开放工位区、休闲协作区、会议室、电话亭、前台、茶水间等。这些区域的空间特征差异很大，不能作为一个整体来提取特征。我们需要把整层平面图拆分为独立的"空间单元"，每个单元作为一个案例进行特征提取。

### 3.2 拆分策略

拆分的依据是**功能区域**，而不是物理墙体。具体策略：

1. **识别功能分区**：LLM 或人工分析整层平面图，识别出所有功能不同的空间区域
2. **标注裁切区域**：为每个区域标注矩形裁切框（bbox），框的边界可能对应实体墙，也可能是功能区域的逻辑分界线
3. **同质性合并**：对于同类型的重复空间（如一层楼有 8 个几乎一样的小会议室），只选取一个代表性单元，在元信息中记录该类型的总数
4. **排除非目标区域**：核心筒、电梯厅、卫生间、走廊等交通/服务空间不纳入拆分

工具支持：`scripts/plan_splitter.py` 实现了"分析模板生成→预览确认→执行裁切"的三步工作流。

### 3.3 子单元边界问题

这是一个需要正视的实操难题。很多共享办公空间内部并没有实体墙进行分隔，所谓的"空间单元"可能只是功能区域的概念划分。这带来两个问题：

**问题一：裁切出的平面图可能不包含完整的墙体围合。**

例如，一个开放工位区可能只有两面实体墙（其中一面有窗），另外两侧是走道或与其他功能区的过渡地带。裁切框画在功能边界上，但那条线并不对应任何物理构造。

处理方式：在手绘平面图时，用**实线表示实体墙，裁切边界（非实体墙的边界）不画线或用虚线表示**。这样 LLM 在阅读平面图时能区分"哪些面有围护、哪些面是开放的"，从而正确判断围合度。

**问题二：缺少墙体是否影响特征提取的准确性。**

影响是有限的，且可以处理：

- **LLM 提取**（38 个特征）：LLM 的判读是语义层面的，它能理解"三面墙一面开放"的空间形态，正常输出围合度、面积、家具等大部分特征。只要平面图的绘制规范清晰（实墙 vs 非墙边界的表达方式明确），LLM 不会因为缺少一面墙而无法工作。
- **OpenCV 平面几何**（5 个特征）：二值化轮廓可能因不闭合而影响形状特征的计算精度，但这 5 个特征（紧凑度、矩形度等）不是壳子聚类的核心参数，影响可接受。
- **Mask2Former + OpenCV 感知**（13 个特征）：完全不受影响，因为它们分析的是透视照像素，与平面图无关。

### 3.4 围合度与窗墙比的联动规则

围合度（enclosure_ratio）是描述空间封闭程度的连续值（0-1），它直接影响窗墙比（window_wall_ratio）的有效性：

| 围合度范围 | 空间特征 | 窗墙比处理 |
|-----------|---------|-----------|
| 0.8-1.0 | 封闭空间，四面（或近似四面）有围护 | WWR 完全有效，正常提取 |
| 0.5-0.8 | 半围合空间，两三面有墙 | WWR 仍提取，但仅描述"有墙面的窗户占比"，后续分析需结合围合度解读 |
| < 0.5 | 开放空间，围护面不完整 | WWR 标记为 -1（无意义），因为"围护面总面积"这个分母本身不成立 |

这条规则需要写入 LLM prompt 模板中，让 LLM 在提取特征时自动遵守。

**关键认知：案例中的围合度 vs 实验中的围合度。** 案例收集阶段如实记录每个空间单元的围合度（可能从 0.2 到 1.0 不等）。但到了 VR 实验阶段，所有原型空间都有完整的围护面（四面墙），围合度通过墙面的通透程度来体现——全实墙=围合度高，大面积玻璃墙或开放式门洞=围合度低。这意味着在 VR 中，"围合度"不是"有没有墙"，而是"墙有多通透"。窗墙比在 VR 实验空间中始终有效。

## 四、平面图绘制规范

### 4.1 手绘平面图的图例约定

采用建筑制图的常规惯例，确保 LLM 能准确识别各建筑元素：

| 图元 | 绘制方式 | 说明 |
|------|---------|------|
| 实体墙 | 粗实线（双线表示厚度更佳） | 空间的物理围护边界 |
| 门 | 弧线（表示开启方向） | 含单扇门和双扇门 |
| 窗户 | 双线/三线（表示玻璃） | 位于墙体上 |
| 家具轮廓 | 细实线 | 桌、椅、沙发、柜等 |
| 非墙边界 | 不画线或虚线 | 仅在拆分子单元时，标记功能区域的逻辑分界线 |

### 4.2 LLM 识别策略

不依赖固定的图例图片，而是在 LLM prompt 中以文字说明嵌入这些约定。LLM 对建筑平面图有较好的先验知识，能识别常规绘法。建议在实际使用前先画几个测试案例验证 LLM 的识别准确率，再根据结果微调 prompt 中的说明。

## 五、样本量与 VR 还原度

### 5.1 案例样本量

预计从 8-12 个共享办公项目中收集约 40 个空间单元案例。这个样本量对于壳子参数聚类是合理的——4-5 个壳子参数维度，40 个样本，每个聚类簇可以有足够的样本支撑。

### 5.2 VR 还原度（中间路线）

VR 原型空间的还原度采用中间路线——精确还原壳子几何，简化但保留关键建筑元素：

| 建筑元素 | 还原策略 | 目的 |
|---------|---------|------|
| 壳子几何（长宽高） | 精确还原 | 壳子是控制变量，必须精确 |
| 窗户 | 简化模型，保留位置、大小和透光性 | 影响采光感受和视野感知 |
| 天花 | 保留高度感，简化造型细节 | 影响空间纵向感知 |
| 灯光 | 保留色温和照度水平，简化灯具形态 | 影响整体氛围感知 |
| 墙面/地面材质 | 使用中性统一材质 | 避免材质成为干扰变量 |

这条路线的核心原则是：让被试能感知空间的基本围护特征（大小、高低、开合、采光），但不会因为壳子本身的装修风格而产生偏好倾向。陈设方案才是实验要操控的变量。

## 六、待验证事项

以下问题需要在实际案例操作中验证：

1. **LLM 对非闭合平面图的识别准确率**：手绘几个包含开放边界的平面图，测试 LLM 能否正确区分实墙和非墙边界，以及围合度的判读是否合理
2. **手绘平面图的绘制一致性**：不同案例之间，手绘风格是否需要更严格的规范化
3. **聚类结果的可解释性**：壳子参数聚类后，得到的原型是否在空间设计层面有直观的可解释意义
4. **40 个样本的聚类稳定性**：样本量是否足够支撑稳健的聚类结果
